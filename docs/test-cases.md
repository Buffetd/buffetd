 - Feature: 通过 Buffet 代理 Origin 请求
    - Scenario: 首次通过 URL 访问外部服务
        - Given
            - 用户手上有一个可以正常访问的外部接口地址（例如某个公开 API 的完整 URL）
            - 系统中还没有这条地址对应的历史访问和缓存记录
        - When
            - 用户在客户端中，通过 Buffetd 前端页面的代理接口 `/api/x/proxy` 填写这个完整 URL 发起访问
        - Then
            - 用户能够顺利拿到与直接访问外部接口一致的业务数据
            - 这次访问会被 Buffet 记录下来，为后续的缓存使用做准备
    - Scenario: 首次通过已配置的数据源（Source）访问外部服务
        - Given
            - 在 Buffet 后台已经为某个外部服务配置好了一条数据源（Source），例如“天气服务”
            - 该数据源下某个接口路径目前还没有被访问或缓存
        - When
            - 用户在系统中选择这条数据源，并指定要访问的路径（例如某个接口地址的相对路径），发起请求
        - Then
            - 用户可以拿到该外部服务的最新业务数据
            - Buffet 会记录这次访问，用于后续的缓存命中
    - Scenario: 使用 URL 访问时命中缓存
        - Given
            - 之前已经通过 Buffet 成功访问过某个完整 URL，并拿到过正常结果
            - 这条 URL 的返回结果已经被 Buffet 缓存
        - When
            - 用户再次通过 Buffet 使用同一个 URL 访问外部服务
        - Then
            - 用户可以明显更快地拿到数据
            - 对外部服务的实际调用次数少于用户访问次数（Buffet 直接返回缓存结果）
    - Scenario: 使用数据源 + 路径访问时命中缓存
        - Given
            - 某个数据源（Source，例如“天气服务”）已经在 Buffet 中存在
            - 之前曾使用这条数据源 + 某条路径成功访问过外部服务，并返回了正确结果
            - 这条数据源 + 路径组合的结果已经被 Buffet 缓存
        - When
            - 用户再次使用相同的数据源 + 路径，通过 Buffet 发起访问
        - Then
            - 用户可以快速拿到与上次一致的业务数据
            - Buffet 不会重复请求外部服务，而是直接返回缓存结果
- Feature: Pool Data
  - Scenario: 访问静态数据的类型时
      - Given
          - 该数据源的某个接口返回的是静态数据结构
      - When
          - 用户通过 Buffet 访问该数据源的静态数据接口
      - Then
          - Buffet 能正确处理并返回静态数据
  - Scenario: 访问动态数据的类型时
      - Given
          - 某个数据源配置了 supportsPool: true
          - 该数据源的某个接口返回的是动态数据结构
      - When
          - 用户通过 Buffet 访问该数据源的动态数据接口
      - Then
          - Buffet 能正确处理并返回动态数据
          - 在数据池充足时，数据池中的数据随机返回
- Feature: 查看 Metrics
  - Scenario: 运营/技术同事查看 Buffet 运行概况
      - Given
          - Buffet 已经上线运行一段时间，期间产生了一定的访问和缓存行为
      - When
          - 运营或技术同事在后台查看 Buffet 的运行监控（Metrics），例如通过管理界面或监控接口
      - Then
          - 能看到 Buffet 当前的整体运行情况概览，例如：
              - 系统大致运行时长
              - 总访问量，以及缓存命中/未命中的情况
              - 已接入的外部服务/数据源（Source）数量
              - 是否存在大量积压的后台任务，可能影响系统稳定性
          - 这些信息可以帮助判断 Buffet 当前运行是否健康，是否需要扩容或调整配置
